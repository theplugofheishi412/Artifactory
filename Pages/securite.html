Gestion des utilisateurs

Artifactory permet de créer plusieurs utilisateurs avec des rôles spécifiques afin de mieux gérer les accès et responsabilités. Chaque utilisateur peut être assigné à un ou plusieurs groupes selon son rôle.

  • Administrateur : Accès complet, peut créer, modifier et supprimer des artefacts et utilisateurs.
  • Développeur : Peut publier et télécharger des artefacts dans les dépôts autorisés.
  • Utilisateur en lecture seule : Accès uniquement pour consulter et télécharger les artefacts.

Rôles et permissions

La définition correcte des rôles et permissions est essentielle pour sécuriser votre instance Artifactory. Il est recommandé de respecter le principe du moindre privilège.

  • Attribuer uniquement les permissions nécessaires à chaque rôle.
  • Éviter les permissions globales, surtout pour les développeurs et utilisateurs standards.
  • Utiliser des groupes pour gérer facilement les droits par équipe ou projet.

Sécurisation des accès

Artifactory offre plusieurs mécanismes pour sécuriser l’accès aux artefacts et protéger vos dépôts.

  • Changer immédiatement le mot de passe par défaut après l’installation.
  • Forcer l’utilisation de HTTPS pour toutes les connexions.
  • Limiter l’accès par adresse IP lorsque cela est possible.
  • Activer l’authentification LDAP ou SSO pour centraliser la gestion des utilisateurs.

Bonnes pratiques DevOps

  • Séparer les dépôts selon les environnements : développement, test, production.
  • Ne jamais supprimer un artefact en production, privilégier l’archivage ou la promotion des artefacts.
  • Activer la journalisation et les audits pour suivre toutes les opérations sur les dépôts.
  • Automatiser la promotion des artefacts via les pipelines CI/CD.
  • Mettre en place des scans de sécurité pour détecter les vulnérabilités dans les dépendances.

Intégration avec les outils CI/CD

Artifactory s’intègre facilement avec les outils de CI/CD tels que Jenkins, GitLab CI, Bamboo ou Azure DevOps. Cette intégration permet :

  • Automatisation du build et du déploiement des artefacts.
  • Gestion des versions et promotion des artefacts selon les pipelines.
  • Suivi complet des dépendances et versions utilisées dans chaque build.
  • Possibilité de créer des dépôts virtuels combinant plusieurs dépôts physiques.

Acronymes courants dans Artifactory

  • CI : Continuous Integration (Intégration Continue)
  • CD : Continuous Delivery / Deployment (Livraison / Déploiement Continu)
  • RBAC : Role-Based Access Control (Contrôle d’accès basé sur les rôles)
  • LDAP : Lightweight Directory Access Protocol (Protocole d’annuaire)
  • SSO : Single Sign-On (Authentification unique)
  • API : Application Programming Interface (Interface de programmation applicative)
