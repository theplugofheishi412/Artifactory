Premiers pas avec Artifactory

Une fois Artifactory installé et accessible via l’interface web, nous pouvons commencer à gérer les artefacts : création de dépôts, publication et récupération des fichiers.

Créer un dépôt

Étape 1 : Accéder à l’administration

Connectez-vous avec un compte administrateur, puis :

  • Administration
  • Repositories
  • Create Repository

Étape 2 : Choisir le type de dépôt

  • Local : artefacts internes
  • Remote : proxy vers un dépôt externe
  • Virtual : regroupement de dépôts

Exemple : Dépôt Maven Local

  • Type : Local
  • Package Type : Maven
  • Repository Key : maven-local

Publier un artefact

Méthode 1 : Upload manuel

Depuis l’interface web :

  • Artifacts
  • Sélectionner le dépôt
  • Deploy
  • Choisir le fichier

Méthode 2 : Upload avec Maven

Exemple de configuration pom.xml :

<distributionManagement>
  <repository>
    <id>artifactory</id>
    <url>http://localhost:8081/artifactory/maven-local</url>
  </repository>
</distributionManagement>

Publication :

mvn deploy

Télécharger un artefact

Les artefacts peuvent être récupérés :

  • Via l’interface web
  • Via Maven / npm / pip
  • Via une URL directe

Exemple Maven :

<repositories>
  <repository>
    <id>artifactory</id>
    <url>http://localhost:8081/artifactory/maven-local</url>
  </repository>
</repositories>

Gestion des versions

Artifactory permet de gérer plusieurs versions d’un même artefact :

  • Versions SNAPSHOT
  • Versions RELEASE
  • Historique des builds

Cela permet de revenir à une version stable à tout moment.

Bonnes pratiques

  • Ne jamais déployer directement en production
  • Utiliser des dépôts distincts (dev / test / prod)
  • Versionner correctement les artefacts
  • Restreindre les droits d’accès
